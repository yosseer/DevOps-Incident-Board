## ğŸ“Š DEPLOYMENT SPECIFICATIONS

### Pods & Replicas
- PostgreSQL: 1 StatefulSet pod (persistent data)
- Backend API: 2 Deployment pods (load balanced)
- Frontend: 2 Deployment pods (load balanced)
- **Total**: 5 pods across all services

### Resources
- **API**: 512Mi memory request, 1Gi max; 250m CPU request, 500m max
- **Frontend**: 128Mi memory request, 256Mi max; 100m CPU request, 200m max
- **PostgreSQL**: 256Mi memory request, 512Mi max; 250m CPU request, 500m max

### Storage
- **Database**: 10Gi PersistentVolumeClaim (ReadWriteOnce)
- **Data**: Persists across pod restarts and node failures

### Networking
- **Frontend Service**: ClusterIP, port 8080 (internal)
- **API Service**: ClusterIP, port 8080 (internal)
- **PostgreSQL Service**: ClusterIP (headless), port 5432 (internal)
- **Frontend Route**: OpenShift Route with TLS edge termination (external HTTPS)

### DNS Service Discovery
- `frontend.incident-board.svc.cluster.local:8080` (internal)
- `api.incident-board.svc.cluster.local:8080` (internal)
- `postgresql.incident-board.svc.cluster.local:5432` (internal)

### External Access
- **Hostname**: `incident-board.apps.<your-cluster-domain>`
- **Protocol**: HTTPS with TLS edge termination
- **Access**: Frontend only (API is internal)
